# 数据分析软件说明

## 图显示与界面工作流优化
- 主图 `FigureCanvasTkAgg` 已固定挂载到右侧独立的 `plot_container`，不再直接挂到根窗口。
- 右侧使用 `ttk.Notebook`，分为：
  - `表格预览`
  - `图形结果`
- 成功出图和失败提示都会自动切到 `图形结果` 页，不会再出现“图算出来了但页面看不到”的情况。
- 图页已经接入 `NavigationToolbar2Tk`，支持缩放、平移和保存。
- `render_plot_message()` 会把失败原因直接画到图页中央，而不是只改状态栏或只弹窗。
- 左侧工作区按 `单文件分析 / 双设备对比 / 高级参数` 分层，主路径和高级项已经分开。

## txt/log 高频文件适配
- 支持两种入口：
  - `选择目录`
  - `选择文件`
- 可直接打开：
  - `txt`
  - `log`
  - `csv`
  - `dat`
- 对 `txt / log` 高频文件，会结合首行内容自动判断：
  - 如果首行更像“时间戳 + 数据”
  - 默认采用“逗号分隔 + 无表头”
- `header=None` 时会自动生成友好列名：
  - 第 1 列：`时间戳`
  - 后续：`数据列01 / 数据列02 / 数据列03 ...`
- 数值列除了能否转数值外，还会额外检查：
  - 有效点数
  - 标准差
  - 唯一值个数
- 恒定列、近恒定列、点数过少列不会进入谱分析候选区。

## 混合时间戳解析修复
- 当前版本统一使用 `parse_mixed_timestamp_series()` 处理高频时间戳。
- 已兼容以下混合形式：
  - `YYYY-MM-DD HH:MM:SS`
  - `YYYY-MM-DD HH:MM:SS.1`
  - `YYYY-MM-DD HH:MM:SS.9`
- 解析前会统一做：
  - `strip()` 清理
  - 空字符串处理
  - `nan / None` 转缺失
- 优先尝试 `format="mixed"`；
  如果当前 pandas 版本不支持，则自动回退到逐元素解析。
- 时间相关链路已经统一走这套解析逻辑，用于：
  - YGAS 解析
  - TOA5 dat 解析
  - 时间范围统计
  - 采样频率估计
  - 双设备时间对齐
- 界面诊断中会显示：
  - 原始行数
  - 有效时间戳数
  - 有效率
  - 低有效率警告

## MODE1 文件识别
- 已支持以下文件画像识别：
  - `YGAS_MODE1_15`
  - `YGAS_MODE1_16`
  - `TOA5_DAT`
- `YGAS_MODE1_15` 条件：
  - 无表头
  - 15 列
  - 首列像时间戳
- `YGAS_MODE1_16` 条件：
  - 首列像递增序号
  - 第二列像时间戳
- 对 MODE1 15 列数据，会优先套用业务列名模板：
  - `时间戳`
  - `通道1`
  - `通道2`
  - `通道3`
  - `通道4`
  - `状态字符`
  - `设备号`
  - `CO2浓度`
  - `H2O浓度`
  - `CO2信号强度`
  - `H2O信号强度`
  - `温度`
  - `压力`
  - `状态寄存器`
  - `校验和`
- 如果识别到 MODE1，默认优先推荐：
  - `CO2浓度`
  - 其次 `H2O浓度`

## 双设备解析与对比汇总表
- 当前双设备主链已经是：
  1. 选择多个 `ygas txt/log`
  2. 选择一个 `dat`
  3. 自动拼接多个半小时 `txt/log`
  4. 与 `dat` 按时间范围和对齐策略对齐
  5. 生成 `comparison dataframe`
- `comparison dataframe` 至少包含：
  - `时间戳`
  - `A_...` 前缀列
  - `B_...` 前缀列
  - `匹配时间差_s`
- 界面新增：
  - `生成对比汇总表`
  - `使用汇总表进行分析`
  - `导出汇总表`
- 生成完成后：
  - 汇总表会直接显示到 `表格预览` 页
  - 状态栏会提示 `已生成对比汇总表：X 行，Y 列`
- 当前最小可用时间范围策略包括：
  - `使用 dat 时间范围`
  - `使用 txt+dat 共同时间范围`
  - 手动开始/结束时间
- 当前最小可用对齐策略包括：
  - `最近邻匹配`
  - `最近邻 + 容差`
  - `重采样后对齐`

## 汇总表驱动的双列频域分析
- `使用汇总表进行分析` 后，comparison dataframe 会被载入当前主分析区。
- 载入后，现有单文件分析区就直接作用于这个“当前主表”，不再区分来源是普通单文件还是对比汇总表。
- 这意味着后续可以直接在汇总表里勾列，例如：
  - `A_CO2浓度`
  - `B_CO2_Avg`
- 然后继续做：
  - 单列 PSD
  - 双列互谱幅值 `|Pxy|`
  - 协谱 `Re(Pxy)`
  - 正交谱 `Im(Pxy)`
- 汇总表模式下，左侧列选择区会额外提示：
  - 可勾选 1 列做 PSD
  - 或勾选 2 列做互谱 / 协谱 / 正交谱
  - 建议优先选择一列 `A_...` 和一列 `B_...`

## 目标谱图模式
- 当前版本新增了 `生成旧软件兼容谱图` 按钮，作为主路径的最终出图入口。
- 目标谱图固定采用以下样式：
  - 多组散点叠加
  - 每组固定颜色
  - `log-log` 坐标
  - 右上角图例
  - 一条公共 `-2/3` 虚线
  - 主页面 `图形结果` 页直接显示
- 目标谱图流程为：
  1. 选择多个 `ygas txt/log`
  2. 选择一个 `dat`
  3. 每个 `ygas` 文件单独作为一个时间窗口
  4. `dat` 按这些时间窗口逐一切片
  5. 每个窗口分别生成 `ygas` 谱和 `dat` 谱
  6. 最后将所有窗口的谱统一叠加到一张图
- 当前主路径优先常用要素：
  - `CO2`
  - `H2O`
  - `温度`
  - `压力`
- 当前目标谱图不会把 `CO2浓度` 和 `CO2信号强度` 混在同一张图里。
- 目标谱图只针对“同一种物理量”出图，例如：
  - `CO2` 只画 CO2 相关列
  - `H2O` 只画 H2O 相关列
  - `温度` 只画温度
  - `压力` 只画压力
- 目标谱图逻辑与汇总表双列分析已经分开：
  - 目标谱图模式：
    每个 `ygas txt/log` 文件都作为一个独立时间窗口
  - `dat` 会按这些窗口逐一切片
  - 每个窗口最终生成两条谱：
    - 该窗口的 `ygas` 谱
    - 同时间窗的 `dat` 谱
- 例如选择 3 个半小时 `ygas` 文件，并且 `dat` 覆盖完整：
  - 最终应得到 3 条 `ygas` 系列
  - 加 3 条 `dat` 系列
  - 共 6 条系列
- 图例标签按“设备类型 + 时间窗口”生成，不再直接使用内部列名前缀 `A_ / B_` 作为图例主名。
- 生成成功后：
  - 自动切到 `图形结果` 页
  - 状态栏提示：`目标谱图已生成：<系列数> 组`
  - 诊断区显示：
    - 当前目标要素
    - ygas 窗口数
    - dat 切片数
    - 最终参与绘图的系列数
    - 每组有效点数
    - 参考线 `-2/3`
- 目标谱图默认不依赖独立窗口，不使用 colorbar，也不使用频率渐变色。

## 目标谱图逻辑修正
- 当前版本已将“目标谱图模式”和“汇总表双列频域分析模式”明确分开：
- `生成旧软件兼容谱图`：
    面向旧软件那张多窗口叠加谱图
  - `生成对比汇总表 / 使用汇总表进行分析`：
    面向后续互谱、协谱、正交谱分析
- 目标谱图按“组”生成，不再对 `comparison_frame` 的 `A_ / B_` 列直接做 PSD。
- 每个被选中的 `ygas` 文件就是一个独立组：
  - 该组自己的 `ygas` 时间序列独立算 PSD
  - `dat` 按该组时间窗切片后独立算 PSD
  - 两个设备各自产生自己的谱，再统一叠加
- 同一种要素会单独出图，不再把信号强度混入同一张目标谱图。
- 如果某些窗口没有成功生成对应谱，诊断区会直接提示：
  - 该窗口被跳过
  - 请检查 `dat` 覆盖范围和要素列映射

## 旧软件兼容目标谱图模式
- 新增独立主链与独立按钮：
  - 主页面按钮为 `生成旧软件兼容谱图`
  - 不再让普通 `批量谱图对比` 冒充旧软件兼容模式
  - `perform_multi_spectral_compare()` 继续用于同名列批量 PSD 对比
  - `run_dual_device_compare()` 继续用于双设备时间序列 / 散点 / PSD / 协谱 / 正交谱
- 旧软件兼容谱图的数据链固定为“按组出谱”：
  1. 用户选择多个 `ygas txt/log` 和 1 个 `dat`
  2. 当前目标要素固定从 `CO2 / H2O / 温度 / 压力` 中选一个，默认 `CO2`
  3. 每个 `ygas` 文件都视为一个组
  4. 每组读取自己的开始/结束时间作为时间窗
  5. `dat` 按该组时间窗单独切片
  6. 每组只生成两条谱：`ygas` 当前要素谱 + `dat` 当前要素谱
  7. 最终把所有保留组叠加到一张 log-log 散点谱图上
- 当前模式显式使用 `ELEMENT_PRESETS` 做要素列映射：
  - `CO2` 只取浓度列，不再混入 `CO2信号强度`
  - `H2O` 只取浓度列，不再混入 `H2O信号强度`
  - 诊断区会直接显示当前要素、`ygas` 实际列和 `dat` 实际列
- 组级 QC 已接入旧软件兼容主链：
  - 默认跳过 `dat` 无覆盖、点数不足、时间窗过短、频点太少、FS 异常的组
  - 默认比较各组 `log10` 中位谱值，自动跳过明显离群组
  - 组预览摘要会显示组标签、起止时间、点数、FS、频点数、使用状态和跳过原因
  - 被自动跳过但允许人工复核的组，可在组级质控列表中手动放开后重跑
- 出图样式固定为旧软件兼容风格：
  - `scatter`
  - `x/y` 双对数坐标
  - 固定分组颜色
  - 无 `colorbar`
  - 右上角图例
  - 公共 `-2/3` 虚线
  - 图标题为 `时间序列谱分析 - <目标要素>`

## 旧软件兼容目标谱图的组级质量控制
- 目标谱图已从“按列直接出谱”收口为“按组出谱”：
  - 每个 `ygas txt/log` 文件对应一个独立组
  - `dat` 会按该组的时间窗单独切片
  - 两侧都基于各自原始时间窗序列独立计算 PSD
- 当前版本新增了组级质量控制（Group QC），默认会自动跳过这些组：
  - `dat` 时间窗覆盖不完整
  - `ygas` 或 `dat` 有效点数不足
  - `ygas` 或 `dat` 有效频点过少
  - 组内估算 `FS` 明显异常
  - 同一要素下，谱整体中位值明显偏离其他组
- 这样可以优先过滤掉明显异常或不完整的时间组，让目标谱图更贴近旧软件参考图。
- `dat` 目标列映射现在会显式固定并写入诊断信息，例如：
  - 目标要素
  - `ygas` 实际列名
  - `dat` 实际列名
  - 最终参与绘图的组数
  - 被跳过的组及原因
- 程序内部已明确切断目标谱图与汇总表链：
  - 目标谱图不再直接对 `comparison_frame` 或 `A_ / B_` 汇总列做 PSD
  - 只允许使用每个组自己的 `ygas` 原始序列和 `dat` 原始时间窗切片
- 高级功能区增加了“旧软件兼容谱图组级质控”列表：
  - 默认被 QC 跳过但仍可人工放开的组，会列在这里
- 选中后重新点击 `生成旧软件兼容谱图`，即可手动保留这些组
- 目标谱图生成后，诊断区会直接显示组预览摘要，包括：
  - 组标签
  - 每组 `ygas/dat` 点数
  - 每组 `ygas/dat` 频点数
  - 当前使用状态（保留 / 手动保留 / 跳过）
  - 跳过原因

## 目标谱图最终收口
- 目标谱图已经真正接入界面上的时间范围设置：
  - `使用 dat 时间范围`
  - `使用 txt+dat 共同时间范围`
  - `手动输入时间范围`
  - 最近时间窗策略
- 当前目标谱图不再把所有能算出 PSD 的组直接入图，而是先经过组级质量控制（Group QC）：
  - 默认自动跳过覆盖不足的组
  - 默认自动跳过有效点数或有效频点不足的组
  - 默认自动跳过采样频率明显异常的组
  - 默认自动跳过相对其他同要素组谱值明显离群的组
- 高级功能区中的“旧软件兼容谱图组级质控”列表可以手动放开被跳过的组：
  - 重新勾选后再次点击 `生成旧软件兼容谱图`
  - 可用于人工复核像 `1430` 这类可疑时间组
- 目标谱图图例命名已改为更贴近旧软件的组标签：
  - `ygas` 侧优先生成类似 `CO2-ZH-1500`
  - `dat` 侧统一生成类似 `2026-03-27 1500 csi`
  - 不再把 `A_ / B_` 列名前缀、`SAMPLE`、`series` 作为主图图例

  - 当前目标要素
  - 实际 `ygas` 列
  - 实际 `dat` 列
  - 分组策略
  - 总组数 / 保留组数 / 跳过组数
  - 预期系列数 / 实际系列数
  - 跳过原因列表

## 目标谱图从列驱动改成组驱动
- 当前版本已明确把“目标谱图模式”和“汇总表模式”拆开：
  - `生成对比汇总表 / 使用汇总表进行分析`
    继续用于时间序列对比、散点一致性、差值/比值、双列互谱、协谱、正交谱。
  - `生成旧软件兼容谱图`
    单独走旧软件兼容的组驱动主链。
- 目标谱图不再对 `comparison_frame` 或 `A_ / B_` 列直接做 PSD。
- 目标谱图的实际数据链为：
  1. 逐个解析被选中的 `ygas txt/log`
  2. 每个 `ygas` 文件就是一个组
  3. 读取该组自己的开始/结束时间
  4. 在 `dat` 中切出同时间窗的原始子序列
  5. 从 `ygas` 当前要素列提取原始序列
  6. 从 `dat` 当前要素列提取原始切片序列
  7. 两侧分别独立计算 PSD
  8. 仅让通过组级 QC 的组进入最终目标谱图
- 目标谱图图例也不再直接使用内部列名：
  - `ygas` 侧采用类似 `CO2-ZH-1500` 的组标签
  - `dat` 侧采用类似 `2026-03-27 1500 csi` 的组标签
- 高级区中的组级质控列表可手动放开被自动跳过的组，重新点击 `生成旧软件兼容谱图` 后生效。
